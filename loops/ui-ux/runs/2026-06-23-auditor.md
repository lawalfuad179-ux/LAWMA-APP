# Auditor Run — 2026-06-23

## Setup
- Dev server started via `mcp__Claude_Preview__preview_start` (lawma-dev → port 3100, hardcoded in `package.json` dev script).
- Viewport: mobile (375x812) — primary target audience is Lagos residents on phones.
- Brand source of truth: `tokens/tokens.css` (CSS vars) + `Brand Assets/Logo/`.

## Pages

### 1. `/` (Landing)
- Hero section is **1136px tall** — content sits below the 812px fold on mobile.
- DOM confirms `<h1>` renders with opacity 1 and Lagos State Government badge image loads (1760x1612). Issue is layout, not animation.
- LandingHeader shows a "Get started" button next to a theme toggle in the top-right — odd grouping. CTA duplicated at the bottom of hero.

### 2. `/login`
- "Sign in" headline ~72px on mobile — visually shouty for a utility action.
- Continue button confirmed disabled at default state (opacity 0.45, bg `rgb(252, 103, 3)`) — that's intentional but easy to mistake for a styling bug.
- Subhead reads "Enter your phone number to sign in" but input placeholder is `you@example.com or 080 12...` — copy mismatch + truncation.
- Theme toggle present on auth screen — wrong altitude.
- Generic dots glyph used as logo — not the LAWMA logo from `Brand Assets/Logo/`.

### 3. `/dashboard` → redirects to `/login` (auth gate). Audited `/onboarding` instead.

### 3'. `/onboarding`
- Strongest of the three. Clear hierarchy, good headline ("Lagos, meet your waste companion"), three feature cards.
- Progress bar reads ~80% filled but label says "Step 1 of 2" — mismatch.
- "Back" button appears between progress bar and headline (inline). Conventional spot is header-left.
- Feature card icons sit inside their own gray squares inside the card — double nesting.
- "LGA" abbreviation used without expansion — residents may not all parse it.
- Theme toggle present here too.

## Cross-page observations
- **Theme toggle is everywhere it shouldn't be.** Landing header, login, onboarding. Belongs in settings/profile only.
- **Generic dots logo used app-wide** instead of the LAWMA logo asset. Brand inconsistency.
- **Next.js dev "N" badge visible** in screenshots — confirm it's hidden in production build.
